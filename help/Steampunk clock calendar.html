Welcome to the Steampunk Clock Calendar Widget.

The Steampunk Clock Calendar VB6/TwinBasic widget is a useful utility displaying a clock /calendar in a decidedly Steampunk fashion on your desktop. It can be resized and placed anywhere on your desktop. It uses minimal resources and helps give your desktop a Steampunk make-over. On modern systems the CPU usage of this widget is negligible less being than 0.9% (i7 6700K @4ghz).  This help file describes just the Clock/Calendar widget and not the other widgets shown above. The others are a mix of VB6/TwinBasic/TwinBasic binaries and javascript desktop widgets that will be converted to VB6/TwinBasic/TwinBasic 'widgets' one at a time as time allows

This version was developed on Windows 7/10 using 32 bit VisualBasic 6 and TwinBasic as a FOSS project, creating a WoW64 widget for the desktop. This version will run on Windows Vista, 7, 8, 10 and 11+. There will also be a separate version for Windows XP and ReactOS. These latter versions will not the option for DPI-awareness, otherwise they will operate exactly the same.

It is open source to allow easy configuration, bug-fixing, enhancement and community contribution towards a set of free-and-useful VB6/TwinBasic utilities that can be created by anyone. The next step is conversion of each to TwinBasic 64 for future-proofing.

This utility is one of a set of steampunk and Steampunk widgets. That you can find them all here on Deviantart: https://www.deviantart.com/yereverluvinuncleber/gallery

I do hope you enjoy using this utility and others. Your own software enhancements and contributions will be gratefully received if you ever choose to contribute.


Installation

When you run the installer a splash screen will appear for 1 brief second, then it will suggest default locations where you can install this utility. When the installation is complete it will run the program. That is it.

To uninstall the program, there is an uninstall.exe that will remove the application for you in the program folder. If you ever want to to reinstall a new version of the program, always uninstall the old version beforehand.

Anti Virus Tools

When you install the Steampunk Clock Calendar Widget for the first time you will need to accept that your anti-virus tool will pause the installation to test the installation program for malware. Due to this program being written in VB6/TwinBasic and being a new program your a/v tool has not encountered, it WILL flag it as being potentially malware. This is a FALSE POSITIVE. This occurs with such tools as Avast or similar, see below.

If you do not feel at ease with running this tool with an exception for malware scans, then I suggest you uninstall it and instead only run software you trust. To help you gain that trust, the source code for this utility is available for you and others to view on github. You can view and edit that code at your leisure and you can even compile the code for yourself. This should give you the confidence that you need. There is a link on the tool's right click menu that will take you straight through to the github page here: https://github.com/yereverluvinunclebert/Steampunk-Volume-Control-VB6/TwinBasic

The a/v tool I have installed for testing on my PC is Avast - that is an a/v tool that many will have installed. The following is what occurs during installation on my system: You download and run the clockCalendarInstaller.EXE which is the installer for this program. When Avast first encounters the file it will test it for malware, it takes approx. 20-30 seconds to complete a scan. Let it finish. It should find no trace of malware. Once done, the clockCalendarInstaller will then perform the installation and when completed it will attempt to run the Steampunk Clock Calendar and place it on your desktop At this point Avast may do another scan to test the Steampunk Clock/Calendar Widget program binary (the Steampunk Clock Calendar.exe file) which will take another 30 seconds or so. When that second scan is complete you should find the Steampunk Clock/Calendar on your desktop running nomally.

Until the Avast databases are updated and it fully trusts the program, Avast may try to scan it every time it is run. This is a pain but it will eventually go away. In the meantime, if you want to prevent your a/v tool from scanning (and being continually intrusive on a trusted program such as this) then you will need to set an exception or an exclusion within Avast (whichever anti-virus tool you use).  Eventually, the creators of the various a/v tools will become familiar with this particular utility and the binary will no longer raise any false positives nor alarms.

This is a new VB6/TwinBasic utility so it is an inevitable occurrence - being a new and unknown binary, it is of course frustrating but it will eventually go away.

Basic Functionality

When you install and run the widget for the first time it will present you with the licence screen and launch the prefs. You must accept the licence terms to proceed further.  The Clock/Calendar will also appear on your main monitor, animating slowly using minimal resources. This widget size can be increased in size (press the CTRL key in combination with the mouse scrollwheel UP/DOWN to resize), animation speed can be changed, opacity/transparency may be set as to the user's discretion. The widget can also be made to hide for a pre-determined period. If the application is already running then you can double-click to run a new instance of the binary and you will have two or more Clock/Calendars running on the desktop. As each Steampunk Clock/Calendar running program is a mere desktop 'Widget', each instance will not appear on the taskbar unless you configure it to do so. Widgets tend to operate underneath all your other programs and sit quietly on the desktop doing their thing.

Alarm Flag (1)

If an alarm is being displayed then an alarm flag will display at this position giving the number of the alarm.

Weekday Flag (2)

If you press the weekday toggle a weekday indicator will pop up to display the day of the week.

Display Console (3)

The display console informs the user of the clock's internal functions as they occur. Think of it as a debug console that displays the inner workings of the program. If you double-click on the screen, it will send a carriage return and roll up the text one line.

Screeen Toggle (4)

The screen toggle controls the display console, press it and the console elements will retract and disappear. Press it again and it will reappear.

Back Screeen Toggle (5)

The back screen can be raised or lowered.

Back Screeen (6)

The back screen when elevated, allows the transparent console display to be viewed on darker backgrounds.

The Alarm Bell (7)

If you press the alam bell it emits a 'ding' but nothing more, however, in 'alarm edit' mode it sets and confirms the current alarm.

The Alarm Switch (8)

If you press the alam switch then it will switch the whole clock into alarm mode and raise the currently selected alarm. It will also free the time slider, allowing you to advance or retard time as you require.

Digital Time Display (9)

The digital time display will show the current time in 12 or 24 hour clock according to your preference. When in alarm mode it will display the time of the currently active alarm.

Date Display (10)

The date display will show the current date in standard dd/mmm/yyyy manner. When in alarm mode it will display the date of the currently active alarm.

Morning/Afternoon (11)

The AM/PM display will show the current time anti or post meridien. When in alarm mode it will display the same for the currently active alarm.

Time Slider (12)

The time slider is only freed after you enter alarm mode by pressing the A toggle first. When freed it will centre itself to point 2, the central point. To advance or retard the alarm time click on the time slider and drag it left and right within the bounds of the tool. The time displayed will advance or retard accordingly.

The Analogue Clock (13)

The analogue clock is just that, an ordinary clock. When in alarm mode it will display the time of the currently active alarm.

The Sound Crank (14)

The sound crank has two positions, up and down. When you click on the sound crank it will toggle to the alternative position. Down means quite and up means loud, as in "crank up the volume".

The Pendulum (15)

The pendulum will swing if you click on the pendulum or the P toggle. Note that the pendulum swing is achieved using maths, ie. code, so that it uses up CPU cycles, possibly 3-4% depending upon the age and power of your CPU. If you want it to use less CPU, click it again to stop.

The Ticking Control (16)

The ticking chain, when pulled produces a regular ticking sound. Pulling it again, will silence the tick.

The Alarm Toggles (17)

The alarm toggles, all five of them are activated by clicking on each. If an alarm has been pre-set then that alarm will be displaye. If there is no alarm set then the clock will display a time of "00:00" and a date of  "01 JAN 1970".

The Set Toggle (18)
Press the Alarm Toggle first to start editing an alarm but then press the set toggle that will save any alarm currently being edited, returning the clock back to clock mode.

The Mute Toggle (19)

The mute toggle silences the whole widget including alarm chimes.

The Locking Nut (20)

The locking nut locks the widget in place upon the desktop. Click it to toggle on/off. This will also store the widget's current X,Y location on the screen, storing these values in the preferences under the Position Tab. If the widget is restarted it will relocate to this position automatically.

Widget Help Toggle (22)

When you select this option the brief one-screen help pop-up will appear.as above.

The Author's Notes  (23)

The author's notes will be revealed when the pulling ring is clicked upon.

The Alarm Clapper (24)

The alarm clapper can be made to move away from the bell when clicked, silencing only the chimes.

The Pendulum Toggle (25)

The pendulum will swing if you click on the pendulum or the P toggle. Note that the pendulum swing is achieved using maths, ie. code, so that it uses up CPU cycles, possibly 3-4% depending upon the age and power of your CPU. If you want it to use less CPU, click it again to stop.

The Weekday Indicator (26)

The weekday indicator provides information about the current day, ie. the day number and name.

Reveal Widget in Windows Explorer.

When you select this option, a file explorer window will open using the folder location where your application is situated. A most useful function to find the application quickly.

Reload Widget (F5).

This option quietly stops and restarts the Steampunk Clock Calendar Widget. On the rare occasion that the Steampunk Clock Calendar Widget has a tiny problem, possibly due to an unforeseen bug - in that case you can simply reset it by reloading the widget. In addition, if the widget has focus then a key-press on the F5 key will also restart the widget. Be aware this is a soft-reload, just initialising and re-running the main software components.

If you want to perform a hard restart, ie. program closing and restarting then you can press SHIFT+R. The program will shutdown and use the restart binary to restart itself.

Fig 08. The Config Prefs pane.

 The configuration panel is the location for optional configuration items. These items change how Steampunk Clock Calendar operates. Configure them to suit your needs and your mode of operation. You may adjust the widget to a percentage of the original size.  As an alternative, you can mouse hover over the widget itself and press the CTRL key in combination with the mouse scrollwheel UP/DOWN to resize. The scroll-wheel resizing direction is determined here, the direction chosen causes the widget to increase in size.  

The option to show the widget in the taskbar is useful as it allows you to hide the taskbar entry for this widget so the taskbar remains uncluttered. Typically widgets sit beneath the user's programs populating the "underside of the desktop", as it were. You may have 40-50 useful widgets running in the background and these operate without polluting the user experience, each being slimline in operation. Therefore, it makes no sense to have the taskbar cluttered by multiple widgets. This option is here just for those that might run one or two such programs and desire to have their widget appearing in the taskbar and acting more as an ordinary Windows program (which it really is).

The DPI awareness toggle requires the widget to be 'hard' started in order to take effect, ie. a program close and re-open. As well as the main program binary there is also a  restart.exe in the app folder that is called by this option, this small program simply restarts the widget itself and no more.

DPI awareness allows the program to respond to the o/s scaling of the desktop, including handling the scaleable fonts so that they remain crisp and clear. Some prefer the old school look of non-DPI aware apps. That option is for you. If you have a smaller screen this option will be best, switched off.

Note: If you toggle between high and low DPI settings then if you have resized the preferences window (by dragging the bottom right corner) it will lose your modified size and revert to the default size.

Technology: 

The utility was created using VB6 and the RC6/Cairo graphic wrapper from Olaf Schmidt. Some brief history: VB6 was once Microsoft's vaunted flagship language but they abandoned it. I used VB6 firstly to prove it can be done and secondly to reacquaint myself with the technology further. Olaf's RichClient 6 upgrades VB6 to incorporate a multi-platform modern graphics engine independent of VB6 native forms.

This documentation will also cover the TwinBasic version, a VB6 alternative that is 100% VB6 language compatible that can also produce 64bit binaries. The utility is being migrated to TwinBasic so there WILL be a version that is future-proof.

This help file will apply to either version as they will all be functionally identical. This program was built upon Windows 7, in my opinion Microsoft's last good desktop operating system with a decent GUI - with no nags, no tiles nor advertising. However, I have recently migrated to developing to Windows 10 on Win10/11+ and above, just as well. It also runs and operates with no issues at all on all later versions of this operating system. It has also been tested on Linux and Mac OSX, both running Wine.
